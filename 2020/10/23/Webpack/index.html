使用

  1. npm init 文件目录初始化

  2. npm install --save -dev，npm i -D，开发环境。

    npm install --save，npm i -S，生产环境。

  3. 安装依赖npm i -D webpack webpack-cli webpack-dev-server babel-loader@7 babel-core babel-preset-env babel-plugin-transform-runtime babel-plugin-transform-decorators babel-plugin-transform-decorators-legacy sass-loader node-sass css-loader style-loader ejs-loader html-webpack-plugin --registry=https://registry.npm.taobao.org/

配置依赖

  • 配置文件webpack.config.js，作为模块导出module.exportse。
  • mode，webpack模式，development/production。
  • path模块，解决文件url问题，通过require导入const path = require('path');
    • resolve方法，相当于命令行cd。
      • __dirname，表示当前webpack配置文件的绝对路径。
      • 目标路径。
  • entry，入口文件对象。
  • output
    • path，打包后路径。
    • filename，入口文件路径。
  • module
    • rules，定义规则。
      • test，匹配以什么结尾的文件。
      • loader，定义处理此类文件的loader。
      • exclude，排除不需要处理的文件。
      • use，使用多个loader的数组，从右向左进行。
      • options
        • name
        • limit
        • esModule
  • plugins，值为数组，配置插件需要require导入。
    • 配置项为数组元素，一般为new XXX()。
  • devServer
    • contentBase，项目路径。
    • compress，启动gzip压缩。
    • open，当打开服务，即开启默认浏览器。
    • host，主机地址。
    • port，服务端口。
