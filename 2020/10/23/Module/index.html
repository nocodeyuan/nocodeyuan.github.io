Javascript模块化

commomJS、AMD、CMD与ES6模块化。

模块化的作用

  1. 防止全局污染。
  2. 规范js脚本依赖与执行顺序。

立即执行函数模块

Node环境（CommomJS）

  • 同步模块加载规范

  • 特点：

    1. js模块相互依赖，需要使用require导入，创建模块实例（立即执行函数）。
    2. 同步加载js脚本。
    3. 缓存机制，每导入一次，都会进行缓存，出现相异再次缓存。

  • require

  • module.exports

AMD

  • 异步模块定义 Asynchronous Module Definition，使用RequireJS，异步加载脚本。
  • 定义模块：define(moduleName,[module],factory)
  • 引入模块：require([module],callback)，依赖前置，依赖完成加载再执行callback。

CMD

  • 通用模块定义 Commom Module Definition，使用SeaJS，依赖就近，按需加载。
  • 定义模块：define(function(require,exports,module){})
  • 引入模块：seajs.use([module路径],function(moduleA,moduleB,moduleC){})

ES6模块化

  • import
  • export default

模块化区别

  • CommomJS与ES6模块化

    1. CommomJS引入的是拷贝的值。

      ES6引入的是值的引用。

    2. CommomJS模块是在运行时加载，

      ES6模块是在编译时加载。
